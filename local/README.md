# local/ — локальный контекст тестировщика в продуктовой команде

Компания работает в матричной структуре: единые стандарты задаёт центральный QA
(руководитель отдела тестирования + Test lead'ы, см. `docs/Operacionnaya-model-QA.md`,
разделы 5, 7, 8) в `docs/` / `checklists/` / `templates/`, а каждая продуктовая команда
адаптирует их под особенности своего домена. Папка `local/` —
место для этой адаптации, у каждой команды/проекта/компании, использующей этот шаблон, — своя.

## Как это работает
- Агент **всегда ищет ответ сначала здесь**, и только если нужного файла нет или в нём
  недостаточно информации — переходит к аналогичному файлу в `docs/` / `checklists/` / `templates/`.
- Структура зеркалит корень проекта:
  - `local/docs/` — вместо/в дополнение к `docs/` (документация вашего продукта, локальные уточнения методологии);
  - `local/checklists/` — вместо `checklists/` (адаптированный чек-лист требований);
  - `local/templates/` — вместо `templates/` (адаптированные шаблоны кейсов/отчётов).
- Файл с **тем же именем**, что и в общей папке (например `local/checklists/requirements-checklist.md`),
  полностью замещает общий для вашей команды.
- Файл с **новым именем** просто добавляется к контексту (например
  `local/docs/api-payments.md` — документация вашего продукта, которой нет в общих стандартах).

## local/ в .gitignore
`local/` в `.gitignore` (кроме README.md в каждой подпапке — они и сама структура папок
приезжают с шаблоном, а содержимое — нет). Из этого следует:
- разные продуктовые команды/проекты/компании, клонирующие этот шаблон, не конфликтуют
  друг с другом через общий репозиторий;
- обновления общих стандартов (`docs/`, `checklists/`, `templates/`) подтягиваются всем
  без переписывания локальных адаптаций;
- локальные адаптации конкретной команды не утекают в общий репозиторий случайно.

## С чего начать
1. Скопируйте нужный файл из `docs/` / `checklists/` / `templates/` в соответствующую
   папку `local/...` с тем же именем.
2. Отредактируйте под свой продукт/команду.
3. Агент начнёт использовать его автоматически — донастраивать ничего не нужно.

Подробнее по каждой подпапке — в её собственном README.md.
