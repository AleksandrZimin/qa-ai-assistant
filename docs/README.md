# docs/ — общие стандарты компании

Эта папка — единая для всей компании методология QA (в git, общая для всех продуктовых
команд). Продуктовая документация и её адаптации под конкретную команду — не сюда, а в
[`local/docs/`](../local/docs/README.md), которая приоритетнее и не попадает в git
(см. `local/README.md`).

Агент ищет ответы **сначала в `local/docs/`, потом здесь** и ссылается на конкретный
файл (и строку, если применимо). Чего нет ни там, ни здесь — того агент не выдумывает,
а спрашивает у вас.

## Рекомендации
- Формат — Markdown (`.md`): его агент читает точнее и дешевле всего.
- Один документ — одна тема, понятное имя файла.
- Большие документы дробите на разделы — агент читает фрагментами и экономит токены.
- Устаревшие версии удаляйте, чтобы агент не ссылался на неактуальное.
- Сюда кладите только то, что действительно общее для всех команд. Продуктовые требования,
  ТЗ, API конкретного продукта — в `local/docs/` вашей команды.
