## 1. Картографические подложки

### БТ-102
**Бизнес-требование:** Система должна предоставлять два источника картографической подложки — внешний картографический сервис (основной) и локальная базовая карта (альтернативный) — с возможностью переключения между ними для всех пользователей.

### ФТ-102.1
Основная подложка — внешний картографический сервис. Альтернативная — локальная карта на основе открытых источников. Переключение доступно всем пользователям системы.

| № | Название | Шаги → Ожидаемый результат | Сценарий |
|---|----------|----------------------------|----------|
| ТК-102.1.1 | Отображение внешней карты по умолчанию | 1. Открыть картографическую подоснову. → Карта отображается на основе внешнего картографического сервиса. Тайлы загружаются корректно. | Позитивный |
| ТК-102.1.2 | Переключение на локальную базовую карту | 1. На карте нажать кнопку переключения подложки. 2. Выбрать локальную карту. → Подложка переключена на локальную. Сущности и слои остаются на прежних местах. | Позитивный |
| ТК-102.1.3 | Переключение обратно на внешнюю карту | 1. Находясь на локальной карте, переключиться обратно на внешний картографический сервис. → Подложка изменена на внешнюю. Положение сущностей на карте не изменилось. | Позитивный |
| ТК-102.1.4 | Отображение карты при недоступности внешнего картографического сервиса | 1. Симулировать недоступность внешнего сервиса (отключить доступ). 2. Открыть картографическую подоснову. → Система автоматически переключается на локальную подложку или выводит сообщение о недоступности сервиса с предложением использовать альтернативу. | Негативный |
| ТК-102.1.5 | Переключение подложки пользователем с минимальными правами | 1. Авторизоваться как пользователь с минимальными правами. 2. Попытаться переключить подложку. → Переключение доступно — функция предусмотрена для всех пользователей. | Позитивный |

## 2. Поиск и отображение адресов на карте

### БТ-103
**Бизнес-требование:** Пользователь должен иметь возможность найти и отобразить адрес или место на картографической подоснове с помощью внешнего сервиса геокодирования.

### ФТ-103.1
Поиск адресов на карте осуществляется через внешний сервис геокодирования. Результаты поиска отображаются на карте. Функция включается в настройках быстрого поиска.

| № | Название | Шаги → Ожидаемый результат | Сценарий |
|---|----------|----------------------------|----------|
| ТК-103.1.1 | Поиск существующего адреса через сервис геокодирования | 1. Включить функцию геокодера в настройках поиска. 2. Ввести корректный адрес в поисковую строку. 3. Выбрать результат. → Карта центрируется на найденном адресе. Метка отображается в указанной точке. | Позитивный |
| ТК-103.1.2 | Поиск несуществующего адреса | 1. Ввести заведомо несуществующий адрес. → Система выводит сообщение «Адрес не найден» или возвращает пустой список результатов. | Негативный |
| ТК-103.1.3 | Поиск адреса при отключённом сервисе геокодирования (по умолчанию) | 1. Не включать функцию геокодера. 2. Попытаться использовать поиск адреса на карте. → Функция недоступна. Система отображает соответствующее сообщение или кнопка поиска геокодером неактивна. | Негативный |
| ТК-103.1.4 | Включение функции геокодирования в настройках поиска | 1. Перейти в настройки быстрого поиска. 2. Включить функцию геокодера. → Функция активирована. Поиск адресов через сервис геокодирования доступен на карте. | Позитивный |

## 3. Управление слоями карты

### БТ-104
**Бизнес-требование:** Система должна поддерживать послойное отображение данных с возможностью управления видимостью и порядком слоёв, включая сущности системы, административные границы и зоны ответственности.

### ФТ-104.1
Каждый слой (сущности системы, границы административных районов, муниципальных образований, зоны ответственности ЭР) управляется отдельно через включение/отключение. Поддерживается фильтрация данных в пределах слоя.

| № | Название | Шаги → Ожидаемый результат | Сценарий |
|---|----------|----------------------------|----------|
| ТК-104.1.1 | Включение слоя сущностей на карте | 1. В панели управления слоями включить слой «Светильники». → Светильники отображаются на карте в соответствии с их геометрией. | Позитивный |
| ТК-104.1.2 | Отключение слоя сущностей на карте | 1. Отключить слой «Светильники» в панели управления. → Светильники скрыты с карты. Другие слои не затронуты. | Позитивный |
| ТК-104.1.3 | Отображение границ административных районов | 1. Включить слой «Границы административных районов». → Границы административных районов региона отображаются на карте. | Позитивный |
| ТК-104.1.4 | Отображение границ муниципальных образований | 1. Включить слой «Границы муниципальных образований». → Границы муниципальных образований отображаются на карте. | Позитивный |
| ТК-104.1.5 | Отображение зон ответственности эксплуатационных районов | 1. Включить слой «Зоны ответственности ЭР». → Зоны ответственности эксплуатационных районов отображаются на карте. | Позитивный |
| ТК-104.1.6 | Фильтрация данных в пределах слоя | 1. Включить слой «Опоры». 2. Применить фильтр по атрибуту (например, по типу опоры). → На карте отображаются только опоры, соответствующие заданному фильтру. | Позитивный |
| ТК-104.1.7 | Одновременное включение нескольких слоёв | 1. Включить одновременно слои «Светильники», «Опоры», «Линии». → Все три слоя отображаются одновременно без конфликтов. | Позитивный |
| ТК-104.1.8 | Отображение карты при отключении всех слоёв | 1. Отключить все активные слои. → Карта отображается с пустой подложкой без сущностей. Подложка не исчезает. | Позитивный |

## 4. Стилизация элементов карты

### БТ-105
**Бизнес-требование:** Различные типы сущностей и их атрибуты должны визуально различаться на карте посредством цветографического представления для облегчения восприятия пространственной информации.

### ФТ-105.1
Разные слои, типы сущностей и значения атрибутов одинаковых сущностей отображаются разными цветами/стилями на карте (например, тип светильника, тип линии).

| № | Название | Шаги → Ожидаемый результат | Сценарий |
|---|----------|----------------------------|----------|
| ТК-105.1.1 | Визуальное различие разных типов светильников на карте | 1. Включить слой «Светильники» с несколькими типами. → Светильники разных типов отображаются разными цветами или иконками в соответствии с настройками стилизации. | Позитивный |
| ТК-105.1.2 | Визуальное различие разных типов линий на карте | 1. Включить слой «Линии» с разными типами. → Линии разных типов (питающая, групповая, сеть) отображаются разными стилями (цвет, тип линии). | Позитивный |
| ТК-105.1.3 | Отображение проектируемых сущностей отдельным цветом | 1. Создать сущность со статусом «проектируемая» и нанести геометрию. → Сущность отображается на карте цветом, отличным от существующих, согласно настройкам стилизации. | Позитивный |
| ТК-105.1.4 | Изменение статуса проектируемой сущности на существующую | 1. Изменить статус сущности с «проектируемой» на «существующую». → Цвет отображения сущности на карте изменился согласно новому статусу. | Позитивный |

## 5. Кластеризация объектов на карте

### БТ-106
**Бизнес-требование:** При отображении большого количества точечных объектов на карте система должна автоматически группировать близко расположенные объекты в кластеры для обеспечения читаемости карты.

### ФТ-106.1
Кластеризация точечных объектов включается автоматически в зависимости от выбранного масштаба. При увеличении масштаба кластеры раскрываются.

| № | Название | Шаги → Ожидаемый результат | Сценарий |
|---|----------|----------------------------|----------|
| ТК-106.1.1 | Отображение кластеров при мелком масштабе | 1. Включить слой с большим количеством точечных объектов (например, «Опоры»). 2. Установить мелкий масштаб карты. → Близко расположенные опоры сгруппированы в кластеры с отображением их количества. | Позитивный |
| ТК-106.1.2 | Раскрытие кластера при увеличении масштаба | 1. На мелком масштабе наблюдать кластер из нескольких объектов. 2. Увеличить масштаб карты. → Кластер раскрывается. Отдельные объекты отображаются индивидуально. | Позитивный |
| ТК-106.1.3 | Нажатие на кластер для приближения | 1. Нажать на кластер на карте. → Карта приближается к области кластера. Объекты начинают отображаться индивидуально. | Позитивный |
| ТК-106.1.4 | Отображение объектов без кластеризации при крупном масштабе | 1. Установить максимальный масштаб карты. → Все объекты отображаются индивидуально без кластеризации. | Позитивный |

## 6. Инструменты создания и редактирования сущностей на карте

### БТ-107
**Бизнес-требование:** Пользователь должен иметь возможность создавать, редактировать, копировать и перемещать сущности и их атрибуты непосредственно через клиентский интерфейс карты с сохранением изменений в БД.

### ФТ-107.1
Инструменты карты обеспечивают: создание точечных/линейных/полигональных объектов, редактирование геометрии, копирование, перемещение сущностей. Изменения сохраняются в БД.

| № | Название | Шаги → Ожидаемый результат | Сценарий |
|---|----------|----------------------------|----------|
| ТК-107.1.1 | Создание точечного объекта на карте | 1. Выбрать инструмент создания точечного объекта. 2. Кликнуть на карте. → Точка размещена. Открывается форма атрибутов сущности. | Позитивный |
| ТК-107.1.2 | Перемещение сущности на карте | 1. Выбрать существующий объект на карте. 2. Перетащить его на новое место. 3. Сохранить. → Геометрия сущности обновлена в БД. На карте объект отображается в новом положении. | Позитивный |
| ТК-107.1.3 | Автоматический подсчёт длины при создании линейного объекта | 1. Создать линейный объект на карте. → В процессе нанесения система отображает текущую длину линии. После сохранения — «Длина по геометрии» заполнена автоматически. | Позитивный |
| ТК-107.1.4 | Автоматический подсчёт площади и периметра при создании полигона | 1. Создать полигональный объект на карте. → В процессе нанесения система отображает текущую площадь и периметр. После сохранения — «Площадь» заполнена автоматически. | Позитивный |
| ТК-107.1.5 | Попытка создания сущности на карте без прав на создание | 1. Авторизоваться как пользователь без прав создания. 2. Попытаться использовать инструмент создания объекта на карте. → Инструмент недоступен или заблокирован. | Негативный |

## 7. Открытие сущности в режимах «Спутник» и «Гибрид» во внешнем картографическом сервисе

### БТ-108
**Бизнес-требование:** Пользователь должен иметь возможность просмотреть местоположение сущности в режимах «Спутник» и «Гибрид» на сайте внешнего картографического сервиса в отдельной вкладке браузера.

### ФТ-108.1
Из карточки сущности или с карты доступна ссылка для открытия местоположения во внешнем картографическом сервисе в режимах «Спутник»/«Гибрид» в новой вкладке.

| № | Название | Шаги → Ожидаемый результат | Сценарий |
|---|----------|----------------------------|----------|
| ТК-108.1.1 | Открытие местоположения объекта во внешнем картографическом сервисе | 1. Выбрать сущность с геометрией на карте. 2. Нажать «Открыть во внешнем сервисе». → В новой вкладке браузера открывается внешний картографический сервис с центрированием на местоположении сущности. | Позитивный |
| ТК-108.1.2 | Открытие во внешнем сервисе для сущности без геометрии | 1. Выбрать сущность без геометрии. 2. Попытаться открыть во внешнем сервисе. → Система выводит сообщение об отсутствии геометрии. Переход недоступен. | Негативный |

## 8. Инструмент печати карты

### БТ-109
**Бизнес-требование:** Пользователь должен иметь возможность распечатать выбранную область карты с активными слоями в различных форматах бумаги, масштабах и ориентации, с возможностью выгрузки в PDF.

### ФТ-109.1
Инструмент печати реализован на локальной картографической подложке. Поддерживаются форматы А0–А4, книжная и альбомная ориентация, масштабы от 1:1 до 1:1000 м/см, выбор DPI, перемещение области мышью. Дополнительные элементы: легенда, дата и время.

| № | Название | Шаги → Ожидаемый результат | Сценарий |
|---|----------|----------------------------|----------|
| ТК-109.1.1 | Открытие инструмента печати карты | 1. Переключиться на локальную подложку. 2. Открыть инструмент печати. → Отображается область выбора для печати с настройками. | Позитивный |
| ТК-109.1.2 | Выбор формата бумаги А4 | 1. В настройках печати выбрать формат «А4». → Область выбора на карте соответствует пропорциям формата А4. | Позитивный |
| ТК-109.1.3 | Выбор формата бумаги А0 | 1. В настройках печати выбрать формат «А0». → Область выбора увеличилась пропорционально формату А0. | Позитивный |
| ТК-109.1.4 | Переключение ориентации листа | 1. Выбрать книжную ориентацию. 2. Переключить на альбомную. → Область выбора перестроилась в соответствии с альбомной ориентацией. | Позитивный |
| ТК-109.1.5 | Выбор масштаба «в 1 см 100 м» | 1. В настройках печати выбрать масштаб «в 1 см 100 м». → Область выбора на карте перестроилась согласно выбранному масштабу. | Позитивный |
| ТК-109.1.6 | Выбор масштаба «в 1 см 1 км» | 1. Выбрать масштаб «в 1 см 1 км». → Область охватывает более крупную территорию согласно масштабу. | Позитивный |
| ТК-109.1.7 | Перемещение области печати мышью | 1. Открыть инструмент печати. 2. Перетащить область печати мышью в другое место. → Область перемещена. Размер области не изменился. | Позитивный |
| ТК-109.1.8 | Добавление легенды на область печати | 1. В настройках печати включить отображение легенды. 2. Сформировать печать. → Легенда отображается в выбранной области на карте. | Позитивный |
| ТК-109.1.9 | Добавление даты и времени на область печати | 1. Включить отображение даты и времени в настройках печати. 2. Сформировать. → Дата и время отображаются в области печати. | Позитивный |
| ТК-109.1.10 | Выгрузка выбранной области в PDF | 1. Настроить область и параметры печати. 2. Нажать «Выгрузить в PDF». → Файл PDF сформирован и скачан. Содержит все активные слои, видимые в области печати. | Позитивный |
| ТК-109.1.11 | Попытка использования инструмента печати на внешней картографической подложке | 1. Убедиться, что активна внешняя картографическая подложка. 2. Попытаться открыть инструмент печати. → Система выводит сообщение о том, что инструмент печати доступен только на локальной подложке. | Негативный |
| ТК-109.1.12 | Печать области с отключёнными всеми слоями | 1. Отключить все слои. 2. Открыть инструмент печати и выгрузить PDF. → PDF содержит только картографическую подложку без сущностей системы. | Негативный |
| ТК-109.1.13 | Выбор разрешения печати (DPI) | 1. В настройках печати выбрать разрешение (например, 300 DPI). 2. Выгрузить в PDF. → PDF сформирован с указанным разрешением. | Позитивный |

## 9. Масштабирование и навигация по карте

### БТ-110
**Бизнес-требование:** Пользователь должен иметь возможность свободно масштабировать и перемещаться по карте для работы с пространственными данными на разных уровнях детализации.

### ФТ-110.1
Карта поддерживает масштабирование колесом мыши, кнопками +/–, а также перемещение перетаскиванием.

| № | Название | Шаги → Ожидаемый результат | Сценарий |
|---|----------|----------------------------|----------|
| ТК-110.1.1 | Масштабирование карты колесом мыши | 1. Прокрутить колесо мыши вперёд на карте. → Карта увеличивается. Объекты на карте остаются на своих местах. | Позитивный |
| ТК-110.1.2 | Уменьшение масштаба карты | 1. Прокрутить колесо мыши назад. → Карта уменьшается. Кластеризация активируется при достижении соответствующего масштаба. | Позитивный |
| ТК-110.1.3 | Перемещение по карте перетаскиванием | 1. Зажать левую кнопку мыши на карте и перетащить. → Карта перемещается в заданном направлении. Объекты остаются на своих координатах. | Позитивный |
| ТК-110.1.4 | Достижение минимального масштаба карты | 1. Уменьшать масштаб до предела. → Дальнейшее уменьшение заблокировано на минимальном допустимом масштабе. | Негативный |
| ТК-110.1.5 | Достижение максимального масштаба карты | 1. Увеличивать масштаб до предела. → Дальнейшее увеличение заблокировано на максимальном допустимом масштабе. | Негативный |
