# local/checklists/ — адаптированные чек-листы вашей команды

Если ваша продуктовая команда тестирует специфичный домен (платежи, интеграции с
конкретным поставщиком и т.п.) и общего чек-листа `checklists/requirements-checklist.md`
недостаточно — положите сюда файл с **тем же именем** (`requirements-checklist.md`) со
своими пунктами. Навык `testing-requirements` будет использовать его вместо общего.

Если файла здесь нет — используется общий чек-лист компании из `checklists/`.
