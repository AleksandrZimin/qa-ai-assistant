# Сохранение контекста сессии

## Зачем
Кэш позволяет продолжить работу в новой сессии, не пересказывая всё заново.
Файлы старше 2 дней удаляются автоматически при старте сессии.

## Алгоритм
1. Собери итог текущей работы — только факты, без пересказа диалога.
2. Сохрани в `.context-cache/YYYY-MM-DD_HH-mm_<тема-кратко>.md` (тема — 2–4 слова латиницей
   или кириллицей через дефис, например `2026-07-10_15-30_кейсы-авторизация.md`).
3. Подтверди тестировщику одной строкой: файл сохранён, что в нём.

## Формат файла
```markdown
# <Тема>
Дата: YYYY-MM-DD HH:mm

## Что делали
- 1–3 пункта: какая задача, по каким требованиям/документам.

## Результаты
- Ссылки на созданные файлы в results/ и ID кейсов в Test IT (если были).

## Решения и договорённости
- Ответы тестировщика на уточняющие вопросы, выбранные трактовки требований.

## Незакрытое
- Открытые вопросы, непокрытые требования, что планировали делать дальше.
```

## Правила
- Файл должен быть коротким (до ~60 строк): это выжимка, а не стенограмма.
- Токены, пароли и содержимое `.env` в кэш не попадают никогда.
- Если в сессии было несколько разных задач — лучше два маленьких файла, чем один смешанный.
