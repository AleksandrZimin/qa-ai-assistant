# local/docs/ — документация вашего продукта

Сюда кладите то, что специфично для вашей продуктовой команды и не является общим
стандартом компании:
- требования/ТЗ/БФТ вашего продукта;
- описание API, доменной модели, справочников, интеграций;
- локальные уточнения к общей методологии из `docs/`, если у вашей команды есть отличия.

Агент ищет ответ здесь в первую очередь, и только потом — в `docs/` (общие стандарты
компании). Файл с тем же именем, что и в `docs/`, полностью замещает общий для вашей команды.

`example-requirements.md` — демо-пример (авторизация), чтобы опробовать агента. Не
относится к реальному продукту; можно удалить и заменить своей документацией.
